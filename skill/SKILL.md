## КРОК 1. Розрахунок 6 індикаторів

**Формули** (застосовувати окремо по кожній громаді + зведено):

```
1. Рівень залучення (%)    = (автори + виборці) / потенційні × 100
2. Охоплення шкіл (%)      = шкіл_учасниць / шкіл_всього × 100   ← заклади освіти, НЕ учні
3. Частка авторів (%)      = автори / навчені × 100
4. Проєктів на 100 авторів = проєкти_на_голосування / автори × 100
5. Якість (%)              = проєкти_на_голосування / розроблені_проєкти × 100
6. Активізація             = виборці / навчені  →  виражати як "1 : X"
```

> **Зверни увагу на формули 2 і 4** — вони відрізняються від деяких старих версій скілу:
> - Інд. 2: знаменник = **заклади освіти**, а не учні.
> - Інд. 4: знаменник = **автори**, а не потенційні учні.

**Зведені показники** = розраховуються по сумі значень усіх громад (не середнє!).

### Еталонні (стандартні) значення

| Індикатор | Еталон |
|---|---|
| 1. Рівень залучення | 72% |
| 2. Охоплення шкіл | 50% |
| 3. Частка авторів | 26,2% |
| 4. Проєктів на 100 авторів | 10,24% |
| 5. Якість | 84,39% |
| 6. Активізація | 1 : 3 |

Якщо користувач надає інші еталонні значення — використовуй їх.

## КРОК 2. Генерація діаграм (matplotlib → PNG)

**Зберігати в `/home/claude/charts/`.** Ці PNG потім вставляються в розділ 2.3 звіту.
Використовуй скрипт `generate_charts_ishgb.py` (повний код — нижче у цьому скілі).

Ключові правила:

- Для > 2 громад: donut → bar chart
- Всі графіки зберігати з `facecolor='white'`, `dpi=150`, `bbox_inches='tight'`

### Чотири обов'язкові діаграми

**Рис. 1 — Подвійні кільця (Donut):** Інд. 1 (рівень залучення) та Інд. 2 (охоплення шкіл).
Зовнішнє кільце = перша громада, внутрішнє = друга, у центрі = зведений %.

**Рис. 2 — Воронка (Funnel bars):** Інд. 3 (частка авторів).
Горизонтальні бари, √-пропорційний масштаб, пунктирна лінія = еталон авторства.

**Рис. 3 — Кластерний стовпчиковий:** Інд. 4 та Інд. 5.
По одному бару на громаду + зведено, горизонтальна пунктирна лінія = еталон.

**Рис. 4 — Bubble (кола):** Інд. 6 (активізація).
`plt.Circle((x, y), radius)` — НЕ scatter. Радіус ∝ √(ratio) × BASE_R.

## КРОК 3. Генерація звіту (.docx) з інтегрованою інфографікою

Діаграми вставляються **безпосередньо в розділ 2.3** основного звіту через `ImageRun`:

```javascript
// Вставка PNG у docx (в розділі 2.3)
function image(filename, w, h) {
  return new Paragraph({
    alignment: AlignmentType.CENTER,
    children: [new ImageRun({
      type: 'png',
      data: fs.readFileSync(filename),
      transformation: { width: w, height: h },
      altText: { title: filename, description: filename, name: filename }
    })]
  });
}

// Типові розміри для A4 з полями 2–2.5 см:
image('/home/claude/charts/chart1_donut.png', 570, 250)
image('/home/claude/charts/chart2_funnel.png', 570, 250)
image('/home/claude/charts/chart3_bars.png',  570, 240)
image('/home/claude/charts/chart4_bubble.png',570, 230)
```

## Типові помилки та як їх уникати

| Помилка | Рішення |
|---|---|
| `ax.pie()` отримує `width` як аргумент | передавати тільки через `wedgeprops=dict(width=…)` |
| `PageNumber` не є конструктором | `new TextRun({ children: [PageNumber.CURRENT] })` |
| Файл не відкривається у Word | обов'язковий крок LibreOffice re-save (Крок 4) |
| Текст обрізається у барах воронки | використовувати MIN_W + MAX_W + sqrt scaling |
| `WidthType.PERCENTAGE` ламає Google Docs | завжди `WidthType.DXA` |
| Булети з unicode `•` | тільки `LevelFormat.BULLET` через numbering config |
| Кирилічне ім'я файлу → помилка LibreOffice | конвертувати через тимчасове ASCII-ім'я |
| `ImageRun` без `altText` → invalid XML | завжди передавати `altText: { title, description, name }` |
| Інд. 2 рахується від учнів | знаменник = **шкіл_всього**, а не учнів |
| Інд. 4 рахується від потенційних | знаменник = **авторів**, а не потенційних |
