# іШГБ: Аналітичний звіт МіО — повний робочий процес

## Що робить цей скіл

Отримує Excel-файл із моніторинговою таблицею іШГБ → розраховує 6 індикаторів
по кожній громаді → порівнює з еталонними значеннями → генерує:
- **Аналітичний_звіт_іШГБ_[рік].docx** — повний звіт (11+ сторінок)
- **Інфографіка_іШГБ_[рік].docx** — 4 діаграми на окремих сторінках

## КРОК 1. Розрахунок 6 індикаторів

**Формули** (застосовувати окремо по кожній громаді + зведено):

```
1. Рівень залучення (%) = (автори + виборці) / потенційні × 100
2. Охоплення (%)        = потенційні / всього_учнів × 100
3. Частка авторів (%)   = автори / навчені × 100
4. Проєктів на 100      = розроблені_проєкти / потенційні × 100
5. Якість (%)           = проєкти_на_голосування / розроблені_проєкти × 100
6. Активізація          = виборці / навчені  →  виражати як "1 : X"
```

**Зведені показники** = розраховуються по сумі всіх громад (не середнє!).

### Еталонні (стандартні) значення

| Індикатор | Еталон |
|---|---|
| 1. Рівень залучення | 72% |
| 2. Охоплення | 50% |
| 3. Частка авторів | 26,2% |
| 4. Проєктів на 100 | 10,24 |
| 5. Якість | 84,39% |
| 6. Активізація | 1 : 3 |

Якщо користувач надає інші еталонні значення — використовуй їх.

## КРОК 2. Генерація звіту (.docx)

### Стиль оформлення (відповідно до PDF-зразка ГО ФРГС)

| Елемент | Параметри |
|---|---|
| Шрифт тіла | Times New Roman 12pt (size=24 half-pt) |
| Шрифт H1 | Times New Roman 13pt Bold AllCaps, колір `#1F3864` |
| Шрифт H2 | Times New Roman 12pt Bold, колір `#1F3864` |
| Поля | top/right/bottom = 2 см (1134 DXA), left = 2,5 см (1417 DXA) |
| Вирівнювання тексту | Justified |
| Нумерація сторінок | внизу праворуч, над горизонтальною лінією |
| Формат | A4 (11906 × 16838 DXA) |

### Обов'язкова структура звіту

```
Титульна сторінка
  - Назва організації Bold Caps Navy зверху по центру
  - «АНАЛІТИЧНИЙ ЗВІТ» великий Bold Navy по центру
  - Підзаголовок + назва проєкту курсивом
  - «підготовлено [ПІБ]» курсивом
  - Рік по центру внизу

ЗМІСТ (текстовий, ручний)

0. ВСТУП
   - Назва проєкту, ГО, партнери, фінансування, терміни
   - Джерела даних

1. САММЕРІ МіО
   1.1 Методологічна рамка (мета, завдання, джерела, методи)
   1.2 Ключові тези та інсайти (5-6 тез Bold + пояснення)

2. РЕЗУЛЬТАТИ: ПОРІВНЯЛЬНИЙ АНАЛІЗ ГРОМАД
   2.1 Порівняльна матриця (таблиця з легендою ▲/▼)
   2.2 Детальний аналіз по кожній громаді
       - окремий підрозділ H3 на кожну громаду
       - по 6 параграфів (один на індикатор):
         Bold "Інд. N — Назва: значення (▲/▼ різниця до еталону)."
         + 1-2 речення інтерпретації
   2.3 Інфографіка результатів іШГБ
       - Панель-таблиця 3×2 з картками індикаторів
       - Далі — вставлені PNG-діаграми з підписами (Рис. 1–4)

3. ВИСНОВКИ ТА РЕКОМЕНДАЦІЇ
   3.1 Загальна оцінка vs еталон (таблиця з ▲/▼)
       + вердикт «УСПІШНЕ / ЧАСТКОВО УСПІШНЕ / ПОТРЕБУЄ ПОКРАЩЕННЯ»
   3.2 Вплив іШГБ на активізацію учнів
       (кількісний, горизонтальний, структурний, інклюзивний виміри)
   3.3 Сильні сторони та вузькі місця (bullets)
   3.4 Практичні рекомендації (по групах: ОМС, координатори, педагоги, донори)
```

### Кольори таблиць

```javascript
const CLR_NAVY  = '1F3864';  // заголовки таблиць (фон + текст білий)
const CLR_PALE  = 'EBF3FB';  // парні рядки
const CLR_GREEN = 'E2EFDA';  // комірки «вище еталону» ▲
const CLR_RED   = 'FCE4D6';  // комірки «нижче еталону» ▼
const CLR_YELLOW= 'FFF2CC';  // комірка «еталон»
```

**Текст ▲** = жирний зелений `#375623`; **▼** = жирний червоний `#833C00`;
**Еталон** = жирний помаранчевий `#7F6000`.

## КРОК 3. Генерація діаграм (matplotlib → PNG)

Зберігати в `/home/claude/charts/`, потім вставляти в окремий docx.

### Діаграма 1–2: Подвійні кільця (Donut)
- Зовнішнє кільце = перша громада, внутрішнє = друга, у центрі = зведений %
- Якщо громад > 2: перемикнутись на звичайний donut для зведеного + легенда
- `wedgeprops=dict(width=0.30, edgecolor='white')` — не передавати `width` у `ax.pie()` напряму

### Діаграма 3: Воронка (Funnel bars)
- Горизонтальні bars з мінімальною шириною MIN_W щоб текст читався
- Масштаб: `√-пропорційний` (sqrt scaling) щоб великий розрив не ховав малі значення
- Пунктир-рамка = еталонний рівень авторства

### Діаграми 4–5: Кластерний стовпчиковий графік
- Пара стовпців на кожну громаду + разом
- Горизонтальна пунктирна лінія = еталон
- Відсоток якості виводити над парою стовпців

### Діаграма 6: Bubble (кола)
- `plt.Circle((x, y), radius)` — НЕ scatter
- Радіус ∝ `√(ratio)` × BASE_R для наочності
- Двостороння стрілка `<->` між проєктним результатом і еталоном

### Вставка PNG у docx
```javascript
new ImageRun({
  data: fs.readFileSync(path),
  transformation: { width: Math.round(emuW/9144), height: Math.round(emuH/9144) },
  type: 'png',
})
```
Типова ширина = 5 800 000 EMU (≈ 16 см на A4).

## Типові помилки та як їх уникати

| Помилка | Рішення |
|---|---|
| `ax.pie()` отримує `width` як аргумент | передавати `width` через `wedgeprops=dict(width=…)` |
| `PageNumber` не є конструктором | використовувати `PageNumberElement` |
| Файл не відкривається у Word | обов'язковий крок LibreOffice re-save |
| Текст обрізається у барах воронки | використовувати MIN_W + MAX_W scaling |
| `WidthType.PERCENTAGE` ламає Google Docs | завжди `WidthType.DXA` |
| Булети з unicode `•` | тільки `LevelFormat.BULLET` через numbering config |
